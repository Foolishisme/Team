# 📘 团队知识库协作指南 (Obsidian + GitHub)

欢迎加入团队知识库！本文档旨在帮助你快速上手 **Obsidian** 并在 **GitHub** 上进行无痛协作。

## 1. 为什么是 Obsidian + GitHub？

- **数据主权**：我们的文档是纯文本（Markdown），不被任何厂商（Notion/飞书）绑定。即使 Obsidian 倒闭，记事本也能打开。
    
- **版本控制**：利用 GitHub，我们可以追溯每一次修改，谁改的、改了什么、什么时候改的，一清二楚。
    
- **双向链接**：不再是死板的文件夹层级，而是通过链接构建知识网络。
    

## 2. ⚡️ Obsidian 简易上手

### 2.1 核心概念

Obsidian 本质上是一个**本地 Markdown 文件阅读器**。

- **仓库 (Vault)**：就是我们电脑上的一个文件夹，里面装着所有的 `.md` 文件。
    
- **笔记 (Note)**：一个 `.md` 文件。
    

### 2.2 必须掌握的语法 (Markdown)

- **标题**：`# 一级标题`，`## 二级标题`
    
- **列表**：`- 内容` 为无序列表，`1. 内容` 为有序列表。
    
- **粗体**：`**重点内容**`
    
- **引用**：`> 引用内容`
    

### 2.3 杀手级功能：双向链接

在文档中输入 `[[`，会立刻弹出库内所有文档的列表。

- **链接已有文档**：输入 `[[项目A]]`，点击即可跳转。
    
- **创建新文档**：输入 `[[一个不存在的概念]]`，按住 `Ctrl/Cmd` 点击它，就会直接创建并进入这篇新文档。
    

## 3. 🧩 必备插件配置

为了保证团队体验一致，请在 `设置` -> `第三方插件` 中确保开启以下插件（我们已预装，请勿随意禁用）：

### 必须开启

1. **Obsidian Git**：协作核心。负责将本地修改同步到 GitHub。
    
2. **Custom Attachment Location**：**【核心配置】** 实现图片自动归类。当你插入图片时，它会自动在当前目录下创建一个与文档同名的文件夹存放图片。
    
    - _配置建议：_ `Location: ./${filename}`。
        
3. **Advanced Tables**：Excel 体验般的表格编辑。如果不装它，Markdown 表格是噩梦。
    
4. **Paste Image Rename**：粘贴图片时自动重命名。我们会配合前一个插件，将图片命名为 `image_时间戳`，保证唯一性。

## 4. 🛡️ Git 协作工作流 (SOP)

**这是本文档最重要的部分。请严格遵守，以避免“合并冲突”带来的痛苦。**

### ✅ 标准作业程序 (The Happy Path)

我们在 Obsidian 中使用 **Obsidian Git** 插件进行操作，但我们**推荐手动触发**关键步骤，而不是完全依赖自动后台同步。

#### 第一步：开工前，先拉取 (Pull) ⬇️

**在你开始任何写作之前**，先获取队友的最新修改。

- 按 `Ctrl/Cmd + P` 打开命令面板。
    
- 输入 `git pull` 并回车。
    
- _右上角提示 "Pulled..." 代表成功。_
    

#### 第二步：专注写作 ✍️

- 正常编辑文档、添加图片。由于开启了 **Custom Attachment Location**，你的图片会自动进到对应的子文件夹。
    

#### 第三步：完工后，提交并推送 (Commit & Push) ⬆️

**在你准备下班或长时间离开前**，上传你的修改。

- 按 `Ctrl/Cmd + P`。
    
- 输入 `git push` (通常插件配置为 "Commit all and push")。
    
- _右上角提示 "Pushed..." 代表成功。_
    

## 5. 🚨 核心禁忌：千万不要上传的文件 (.gitignore)

**【极其重要】** 为了防止所有人的配置互相打架，必须在仓库根目录创建一个名为 `.gitignore` 的文件，内容如下：

```
# 排除 Obsidian 内部配置文件
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.obsidian/graph.json
.obsidian/appearance.json
.obsidian/hotkeys.json

# 排除系统缓存
.DS_Store
Thumbs.db

# 排除插件索引
.obsidian/plugins/obsidian-git/
```

## 6. 🚨 危机处理：遇到冲突 (Conflict) 怎么办？

如果你在 Push 时报错，或者文档里出现了 `<<<<<<< HEAD` 这种奇怪的符号，说明发生了**合并冲突**。

**不要在 Obsidian 里死磕，请按以下步骤处理：**

1. **保持冷静**，数据没丢。
    
2. **使用 VS Code** 打开此文件夹。
    
3. VS Code 会高亮显示冲突部分，选择 `Accept Current` (保留我的) 或 `Accept Incoming` (保留队友的)。
    
4. **保存文件**并在 VS Code 中完成一次 `Commit` 和 `Sync`。
    

## 7. 命名与文件规范 (House Rules)

1. **图片附件**：通过插件自动管理。如果手动移动了文档，记得检查图片路径是否失效。
    
2. **文件命名**：尽量使用英文或拼音命名文件名（如 `Project_Kickoff.md`），兼容性最好。
    
3. **原子化**：一个文档只讲一件事。
    

> **💡 聪生活提示**：系统化执行的第一步是规范化存储。通过插件自动化管理附件文件夹，我们消除了“文件找不着”的信息差，极大降低了维护成本。
